Universal Commerce Protocol overview

How a chat request becomes a checkout

A buyer does not move once through a funnel. They ask, compare, refine, add to cart, and loop through tool calls until the checkout path is clear.

AI Agent conversation
Each turn can trigger another search, detail lookup, cart update, or checkout action.
1

Buyer asks the AI

Intent, budget, style, destination, and image signals enter the conversation.

2

Agent calls tools

The MCP connector turns conversation context into UCP-ready requests.

3

Catalog finds matches

Products, images, prices, variants, and merchant signals come back as data.

4

Cart or checkout updates

The agent prepares line items, buyer context, totals, and the next status.

5

Buyer reviews or refines

The buyer can approve, ask for changes, or return to step 1 with better context.

5 can return to 1 many times before the buyer is ready to finish.
Catalog answers "what should I buy?"

The agent can search across Shopify-powered merchants, narrow by destination, and show product details without sending the buyer across many storefronts.

Checkout answers "where do I safely pay?"

The merchant keeps checkout rules and payment trust. The agent helps prepare the session, then hands off when buyer review or payment is needed.

Takeaway: UCP turns shopping chat into an iterative commerce loop while merchants keep product truth, checkout policy, and payment trust.

UCP responsibility model

Four actors split trust across the purchase

UCP coordinates commerce without making one system own every responsibility. The platform drives the conversation, the merchant owns the sale, and payment credentials remain separated from payment processing.

P

Platform

The AI host or agent client that understands buyer intent, calls tools, presents choices, and coordinates the journey.

↔ UCP protocol
M

Merchant

The seller that owns product truth, checkout policy, fulfillment, risk decisions, and the resulting order.

CP

Credential Provider

A wallet such as Shop Pay, Google Pay, or Apple Pay that releases a buyer-authorized payment credential.

↔ AP2 / payment token
PSP

Payment Service Provider

The processor that authorizes and settles the transaction for the merchant, such as Shopify Payments or Stripe.

P→CP
Platform requests a credential The buyer authorizes the wallet before any reusable payment secret reaches the agent.
M→PSP
Merchant submits the charge The merchant and PSP keep authorization, fraud checks, capture, and settlement in their trust boundary.

Checkout status state machine

Status tells the agent who must act next

Every Checkout MCP response carries a status. The agent updates recoverable information, hands the buyer to continue_url when escalation is required, and completes only when the checkout is ready.

General access requires_escalation plus continue_url is the normal handoff path for most agent integrations.
Eligible completion Only call complete_checkout when the current state is ready_for_complete and usable payment state exists.
Terminal states completed and canceled end the session. Refresh with get_checkout after a hosted buyer handoff.
Rule: Never guess the next action from the tool name alone—read the returned status and messages. See the detailed state model.

Implementation FAQ

Six distinctions prevent common integration mistakes

Expand each answer for the practical boundary. The maintained source, full wording, and Shopify documentation links live in tips.md.

Are Global Catalog metadata fields standard UCP, and can an agent change them?

No. dev.shopify.catalog.global is a Shopify extension that adds fields, including ML-inferred metadata, to the base UCP Catalog capability.

The public interface has no dynamic metadata override. Saved Catalog inputs constrain search results; they do not rewrite attributes, top_features, or unique_selling_points.

Are Shop Pay Wallet and the UCP Shop Pay handler the same integration?

No. Shop Pay Wallet adds UI and payment-request sessions to a commerce checkout. The UCP handler is a delegated-payment flow using an agent client_id and a single-use Shop Token.

Implementing either surface does not configure or expose the other.

Can ECP be implemented by a Remote MCP server alone?

No. The agent host must load continue_url in a web view, exchange JSON-RPC messages, and implement delegated native UI.

An MCP server can return the URL, but it cannot add those capabilities to an arbitrary AI client.

Do I need to create my own Catalog and pass a Catalog ID?

No. Global Catalog searches eligible Shopify merchants by default. Saved Catalog is optional for repeated scope and filters.

When used, pass its saved_catalog_slug, not a Shopify GID. API credentials and product IDs are separate identifiers.

Can an agent cache Catalog results or product images?

No. Shopify says not to cache Catalog search results, and product images must be rendered in real time with the related merchant listing.

Token and endpoint caching are different because they do not replay product data.

Should a single-merchant agent use Global or Storefront Catalog?

Use Storefront Catalog for one known merchant. Use Global Catalog for cross-merchant discovery, comparisons, and recommendations.

Both implement UCP Catalog, but their scope, authentication, and Shopify extension fields differ.

Keep it current: Treat tips.md as the detailed source when Shopify surfaces evolve.
